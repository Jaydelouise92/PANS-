## 2025-02-18 - Form Field Helper Label Disassociation
**Learning:** In React components featuring separate custom header/label sub-components (like `<Field />`), standard layout flow can lead to rendering inputs completely detached from their labels and validation error spans. Screen readers are unable to link the label and error messages with the input unless explicit `htmlFor`/`id` associations, `role="alert"`, and matching `aria-describedby` linkages are programmed on the actual `<input>`/`<textarea>` siblings.
**Action:** Always refactor custom label headers into container elements (like `div` instead of wrapping standard `<label>` tags over unconnected text blocks), use standalone labels with explicit `htmlFor={id}`, and bind inputs using the matching `id`, `aria-invalid`, and `aria-describedby` associated to the helper's generated error ID.
